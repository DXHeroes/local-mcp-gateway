## Feature Explanations

### Key Features Explained

**Tool Aggregation**: Combine tools from multiple MCP servers into a single endpoint.

**Server Prefixing**: Automatic namespacing to prevent tool name conflicts (e.g., `github__create_issue`).

**DCR (Dynamic Client Registration)**: Automatic OAuth client registration without manual app creation.

**Custom TypeScript**: Run custom TypeScript modules as MCP servers within the gateway process.

**Air-gapped**: Deploy in environments with no internet connectivity.

**High Availability**: Multi-node deployment with automatic failover.

## Upgrade Paths

### Free → Pro

Triggers:
- Need more than 1 profile
- Need more than 3 servers
- Hit 1K daily request limit
- Need external/custom servers

### Pro → Team

Triggers:
- Multiple team members
- Need shared profiles
- Need audit logs
- Compliance requirements

### Team → Enterprise

Triggers:
- SSO requirement
- SLA requirement
- >5 team members
- Custom compliance needs
